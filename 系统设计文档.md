# 秒杀系统设计文档

## 1. 系统架构设计

### 1.1 架构概览

秒杀系统采用微服务架构，将系统拆分为多个独立服务，通过API网关进行统一入口管理。

```

│                   (Web / Mobile App)                         │
└────────────────────────┬────────────────────────────────────┘

        ┌────────────────┼────────────────┬──────────────┐
```

### 1.2 服务拆分

#### 1.2.1 用户服务 (User Service)
- **职责**: 用户注册、登录、认证、个人信息管理
- **核心功能**:
  - 用户注册与登录
  - JWT Token生成与验证
  - 用户信息查询与更新
  - 用户黑名单管理（防止恶意秒杀）

#### 1.2.2 商品服务 (Product Service)
- **职责**: 秒杀商品信息管理
- **核心功能**:
  - 秒杀商品发布与下架
  - 商品详情查询
  - 秒杀活动配置（开始时间、结束时间、价格）
  - 商品分类与搜索

#### 1.2.3 订单服务 (Order Service)
- **职责**: 订单生成、支付、发货管理
- **核心功能**:
  - 订单创建与确认（调用库存服务扣减库存）
  - 订单状态管理（待支付、已支付、已发货、已完成）
  - 订单查询与历史记录
  - 订单取消与退款处理（调用库存服务回滚库存）

#### 1.2.4 库存服务 (Inventory Service)
- **职责**: 库存管理与扣减（秒杀系统核心）
- **核心功能**:
  - 库存初始化
  - 库存查询
  - 库存扣减（原子性操作，被订单服务调用）
  - 库存回滚（订单取消时，被订单服务调用）
  - 超卖防护

## 5. 核心设计要点

### 5.1 服务职责划分

**关键原则：单一职责原则 (Single Responsibility Principle)**

| 服务 | 职责 | 对外API | 内部API |
|------|------|--------|--------|
| **库存服务** | 库存数据管理与原子性扣减 | 查询库存 | 扣减库存、回滚库存 |
| **订单服务** | 订单流程管理 | 创建订单、查询订单 | 调用库存服务扣减库存 |

**为什么这样设计？**

1. **库存服务不直接对外暴露扣减接口**
   - `/api/v1/inventory/deduct` 是内部服务间调用接口
   - 需要服务认证（service-token），不允许客户端直接调用
   - 防止绕过订单流程直接扣减库存

2. **订单服务是秒杀的入口**
   - 客户端只调用 `POST /api/v1/orders` 创建订单
   - 订单服务内部调用库存服务扣减库存
   - 保证订单和库存的一致性

3. **流程示意**
```
客户端请求
    ↓
POST /api/v1/orders (订单服务)
    ↓
订单服务调用库存服务
    ↓
POST /api/v1/inventory/deduct (库存服务，内部调用)
    ↓
库存扣减成功/失败
    ↓
订单服务创建订单或返回错误
    ↓
返回结果给客户端
```

### 5.2 库存扣减方案

**问题**: 高并发下如何防止超卖？

**解决方案**:
1. **Redis + Lua脚本**: 原子性扣减库存
2. **乐观锁**: 数据库层面的并发控制
3. **库存预热**: 秒杀前将库存加载到Redis

```lua
-- Redis Lua脚本示例
local key = KEYS[1]
local decrement = tonumber(ARGV[1])
local current = redis.call('GET', key)
if current and tonumber(current) >= decrement then
  return redis.call('DECRBY', key, decrement)
else
  return -1  -- 库存不足
end
```

### 5.3 秒杀流程

```
1. 用户请求秒杀 (POST /api/v1/orders)
   ↓
2. API网关限流检查
   ↓
3. 用户认证与黑名单检查
   ↓
4. 订单服务调用库存服务扣减库存
   ↓
5. 库存扣减成功 → 生成订单 → 消息队列异步处理
   ↓
6. 消费者处理订单（支付、发货等）
   ↓
7. 库存扣减失败 → 返回秒杀失败
```

### 5.4 防止恶意秒杀

#### 6.5.1 库存服务 (inventory_service.py)

```python
# app/services/inventory_service.py
from sqlalchemy.orm import Session
from app.models.inventory import Inventory
from app.cache.redis_client import redis_client

async def deduct_inventory(
    db: Session,
    product_id: str,
    quantity: int,
    order_id: str
) -> dict:
    """
    扣减库存（原子性操作）
    
    1. 先尝试从Redis扣减（快速路径）
    2. 如果Redis中没有，从数据库读取并同步到Redis
    3. 数据库层面使用乐观锁保证并发安全
    """
    
    # 步骤1: 尝试从Redis扣减
    redis_key = f"inventory:{product_id}"
    result = await redis_client.decrby(redis_key, quantity)
    
    if result < 0:
        # 库存不足，回滚
        await redis_client.incrby(redis_key, quantity)
        return {"success": False, "message": "库存不足"}
    
    # 步骤2: 数据库层面扣减（使用乐观锁）
    inventory = db.query(Inventory).filter(
        Inventory.product_id == product_id
    ).first()
    
    if not inventory or inventory.available_inventory < quantity:
        # 数据库库存不足，回滚Redis
        await redis_client.incrby(redis_key, quantity)
        return {"success": False, "message": "库存不足"}
    
    # 使用乐观锁更新
    old_version = inventory.version
    inventory.available_inventory -= quantity
    inventory.reserved_inventory += quantity
    inventory.version += 1
    
    db.commit()
    
    return {
        "success": True,
        "product_id": product_id,
        "deducted_quantity": quantity,
        "remaining_inventory": inventory.available_inventory,
        "order_id": order_id
    }

async def rollback_inventory(
    db: Session,
    product_id: str,
    quantity: int
) -> dict:
    """回滚库存（订单取消时调用）"""
    
    redis_key = f"inventory:{product_id}"
    await redis_client.incrby(redis_key, quantity)
    
    inventory = db.query(Inventory).filter(
        Inventory.product_id == product_id
    ).first()
    
    if inventory:
        inventory.available_inventory += quantity
        inventory.reserved_inventory -= quantity
        db.commit()
    
    return {"success": True, "message": "库存已回滚"}
```

#### 6.5.3 订单服务 (orders.py)

```python
# app/api/v1/orders.py
from fastapi import APIRouter, Depends, HTTPException, status
from sqlalchemy.orm import Session
from app.db.database import get_db
from app.models.order import Order
from app.models.user import User
from app.utils.jwt_utils import get_current_user
import httpx

router = APIRouter()

# 库存服务的URL（应该从配置读取）
INVENTORY_SERVICE_URL = "http://inventory-service:8000"
SERVICE_TOKEN = "your-service-token"

@router.post("")
async def create_order(
    request: dict,
    db: Session = Depends(get_db),
    current_user: User = Depends(get_current_user)
):
    """
    创建订单（秒杀核心流程）
    
    1. 调用库存服务扣减库存
    2. 库存扣减成功 → 创建订单
    3. 库存扣减失败 → 返回错误
    """
    
    product_id = request["product_id"]
    quantity = request.get("quantity", 1)
    
    # 步骤1: 调用库存服务扣减库存
    async with httpx.AsyncClient() as client:
        try:
            response = await client.post(
                f"{INVENTORY_SERVICE_URL}/api/v1/inventory/deduct",
                json={
                    "product_id": product_id,
                    "quantity": quantity,
                    "order_id": f"O_{current_user.id}_{int(time.time())}"
                },
                headers={
                    "Authorization": f"Bearer {SERVICE_TOKEN}"
                },
                timeout=5.0
            )
            
            if response.status_code != 200:
                raise HTTPException(
                    status_code=status.HTTP_409_CONFLICT,
                    detail="库存不足，秒杀失败"
                )
            
            deduct_result = response.json()
            order_id = deduct_result["data"]["order_id"]
            
        except httpx.RequestError:
            raise HTTPException(
                status_code=status.HTTP_503_SERVICE_UNAVAILABLE,
                detail="库存服务暂时不可用"
            )
    
    # 步骤2: 创建订单记录
    order = Order(
        order_id=order_id,
        user_id=current_user.id,
        product_id=product_id,
        quantity=quantity,
        total_price=request.get("total_price", 0),
        status="pending_payment",
        payment_method=request.get("payment_method")
    )
    
    db.add(order)
    db.commit()
    db.refresh(order)
    
    return {
        "code": 0,
        "message": "订单创建成功",
        "data": {
            "order_id": order.order_id,
            "user_id": order.user_id,
            "product_id": order.product_id,
            "quantity": order.quantity,
            "total_price": order.total_price,
            "status": order.status,
            "created_at": order.created_at
        }
    }

@router.get("/{order_id}")
async def get_order(
    order_id: str,
    db: Session = Depends(get_db),
    current_user: User = Depends(get_current_user)
):
    """获取订单详情"""
    
    order = db.query(Order).filter(

    if not order:
        raise HTTPException(

        )

        "code": 0,
        "message": "获取成功",
        "data": {
            "order_id": order.order_id,
            "user_id": order.user_id,
            "product_id": order.product_id,
            "quantity": order.quantity,
            "total_price": order.total_price,
            "status": order.status,
```
